class BrowserType Accessed via chromium / firefox / webkit — launches or connects to a browser
launch(options?)Promise<Browser>
Launches a new browser instance. Returns a Browser object. This is the most common way to start a browser in scripts.
ParameterTypeDescription
options.headlessboolean optionalRun without UI. Default: true
options.slowMonumber optionalSlow down each action by N ms — useful for debugging
options.argsstring[] optionalExtra browser CLI flags e.g. ["--disable-gpu"]
options.executablePathstring optionalPath to custom browser executable
options.timeoutnumber optionalMax ms to wait for browser to start. Default: 30000
options.devtoolsboolean optionalOpen DevTools panel automatically (Chromium only)
typescript
import { chromium, firefox, webkit } from "@playwright/test";

// Headless (default)
const browser = await chromium.launch();

// Headed — see the browser window
const browser = await chromium.launch({ headless: false });

// Slow motion — for demos or debugging
const browser = await chromium.launch({ slowMo: 500 });

// Firefox
const browser = await firefox.launch({ headless: false });

// Safari (WebKit)
const browser = await webkit.launch();
launchPersistentContext(userDataDir, options?)Promise<BrowserContext>
Launches browser with a persistent user profile — cookies, localStorage, and login sessions survive between runs. Returns a BrowserContext directly (not a Browser).
ParameterTypeDescription
userDataDirstringPath to folder where browser profile data is stored
options.headlessboolean optionalRun headless. Default: true
options.viewportobject optional{ width, height }
options.localestring optionalBrowser locale e.g. 'en-GB'
typescript
// Login once, profile is saved to disk
const context = await chromium.launchPersistentContext(
  './user-data',
  { headless: false }
);
const page = await context.newPage();
await page.goto('https://app.com/login');
// Login manually once — next run is already logged in
await context.close();
connect(wsEndpoint, options?)Promise<Browser>
Connects to a remote Playwright server over WebSocket. Used for cloud grids, Docker containers, or browser farms.
ParameterTypeDescription
wsEndpointstringWebSocket URL of the Playwright server e.g. "ws://localhost:9222"
options.timeoutnumber optionalConnection timeout in ms
options.headersobject optionalHTTP headers to include in handshake (auth tokens)
typescript
// Connect to remote Playwright server
const browser = await chromium.connect('ws://remote-server:9222');
const page = await browser.newPage();
await page.goto('https://example.com');
await browser.close();
connectOverCDP(endpointURL, options?)Promise<Browser>
Connects to an existing Chrome/Edge instance via Chrome DevTools Protocol (CDP). Only works with Chromium. Use when you want to attach to a running browser.
ParameterTypeDescription
endpointURLstringCDP endpoint URL e.g. "http://localhost:9222"
typescript
// Start Chrome with: --remote-debugging-port=9222
const browser = await chromium.connectOverCDP('http://localhost:9222');
const contexts = browser.contexts();
const page = contexts[0].pages()[0]; // grab existing tab
executablePath()string
Returns the path to the browser executable that Playwright will use. Synchronous.
typescript
console.log(chromium.executablePath());
// e.g. '/home/user/.cache/ms-playwright/chromium-1091/chrome-linux/chrome'
name()string
Returns the browser name: 'chromium', 'firefox', or 'webkit'. Synchronous.
typescript
console.log(chromium.name()); // 'chromium'
console.log(firefox.name());  // 'firefox'
console.log(webkit.name());   // 'webkit'
class Browser Represents a browser instance. Created by BrowserType.launch()
newContext(options?)Promise<BrowserContext>
Creates an isolated browser session. Each context has its own cookies, localStorage, and auth state — like separate incognito windows.
ParameterTypeDescription
optionsobject optionalviewport, storageState, locale, geolocation, permissions, httpCredentials, extraHTTPHeaders, baseURL, ignoreHTTPSErrors
typescript
// Basic context
const context = await browser.newContext();

// With options
const context = await browser.newContext({
  viewport: { width: 1280, height: 720 },
  storageState: 'auth.json',   // pre-logged-in state
  locale: 'en-GB',
  baseURL: 'https://myapp.com'
});
newPage()Promise<Page>
Creates a new page in a default context. Shortcut when you don't need context isolation.
typescript
const page = await browser.newPage();
await page.goto('https://example.com');
contexts()BrowserContext[]
Returns all open BrowserContext instances for this browser. Synchronous — no await needed.
typescript
const allContexts = browser.contexts();
console.log(`Open contexts: ${allContexts.length}`);
close()Promise<void>
Closes the browser and all its contexts and pages. After this call the browser instance is unusable.
typescript
await browser.close(); // closes ALL contexts + pages inside
isConnected()boolean
Returns true if browser process is still running. Useful in remote/CDP connections.
typescript
if (browser.isConnected()) {
  console.log('Browser is alive');
}
version()string
Returns the browser version string synchronously.
typescript
console.log(browser.version()); // e.g. "120.0.6099.109"

Creates a Locator using CSS, XPath, text, or role selectors. This is the recommended way to find elements.

page.locator('text=Sign In')         // Text
page.locator('[data-testid="btn"]')  // Test ID

Finds element by ARIA role. Most accessible and recommended locator strategy.
typescript
page.getByRole('button', { name: 'Save' }).click();
page.getByRole('textbox', { name: 'Email' }).fill('a@b.com');
page.getByRole('heading', { level: 1 });

Convenience locators — find by visible text, label, placeholder attribute, or data-testid.

page.getByPlaceholder('Search').fill('Playwright');
page.getByTestId('login-btn').click();
Dialogs (Alerts)
on('dialog', handler)void
page.on = listens every time dialog fires (all future dialogs). page.once = handles only the next dialog then removes itself. Always register handler before the action that triggers the alert.
typescript
// page.on — handles ALL dialogs for lifetime of page
page.on('dialog', dialog => {
  console.log(dialog.message()); // 'Are you sure?'
  dialog.accept();               // press OK
});

// page.once — handles ONLY the next dialog, then auto-removes
page.once('dialog', dialog => dialog.dismiss()); // press Cancel
await page.click('#delete-btn'); // triggers the alert

Presses a keyboard key while the element is focused. Good for Enter, Tab, Escape on a specific element.

Chaining & Filtering
filter(options)Locator
Narrows the matched set by text content or by a nested locator that must exist inside each match.
ParameterTypeDescription
options.hasTextstring | RegExp optionalOnly keep elements containing this text
options.hasLocator optionalOnly keep elements that contain this locator
typescript
// Keep only rows containing "Chennai"
await page.locator('tr').filter({ hasText: 'Chennai' }).click();

// Keep rows that have a checked checkbox inside
await page.locator('tr').filter({
  has: page.locator('input[type=checkbox]:checked')
});
and(locator) / or(locator)Locator
and() = element must match BOTH locators. or() = element matching either locator.

await page.getByRole('button')
  .and(page.getByText('Save'))
  .click();

helper Keyboard Accessed via page.keyboard — global keyboard actions
press(key, options?)Promise<void>
Presses a key — fires keydown → keypress → keyup all in one call. Supports key combinations with + separator. Use this for shortcuts.
ParameterTypeDescription
keystringKey name: 'Enter', 'Tab', 'Escape', 'ArrowDown', 'Control+A', 'Shift+Tab', 'F5'
options.delaynumber optionalDelay between keydown and keyup in ms
typescript
await page.keyboard.press('Enter');
await page.keyboard.press('Control+A');  // Select All
await page.keyboard.press('Control+C');  // Copy
await page.keyboard.press('Shift+Tab');  // Reverse tab
await page.keyboard.press('Tab');         // Move focus forward
type(text, options?)Promise<void>
Types a string character by character into the focused element — fires real key events per character. Affects whatever element currently has focus.
typescript
await page.locator('#email').click(); // focus first
await page.keyboard.type('Hello World');
await page.keyboard.type('Slow typing', { delay: 80 });
down(key)Promise<void>
Fires only the keydown event and holds the key. Use with up() to simulate a held key (e.g. holding Shift while clicking).
typescript
// Hold Shift, click multiple items for multi-select
await page.keyboard.down('Shift');
await page.locator('li').nth(3).click();
await page.keyboard.up('Shift');
up(key)Promise<void>
Fires only the keyup event — releases a key previously held with down().
typescript
await page.keyboard.down('Control');
await page.keyboard.press('KeyA');  // Ctrl+A while held
await page.keyboard.up('Control'); // release
insertText(text)Promise<void>
Inserts text directly without firing keyboard events. Useful for pasting special characters or emoji that can't be typed key-by-key.
typescript
await page.keyboard.insertText('こんにちは'); // Japanese text
await page.keyboard.insertText('🚀 Launch!'); // Emoji

typescript

network Route Intercept and control a matched network request
fulfill(options)Promise<void>
Responds to the request with mock data — never hits the real server. Perfect for API mocking.
ParameterTypeDescription
options.statusnumber optionalHTTP status code. Default: 200
options.bodystring | Buffer optionalResponse body
options.contentTypestring optionale.g. 'application/json'
options.jsonobject optionalShortcut: auto-serializes object and sets content-type
typescript
await page.route('**/api/products', route =>
  route.fulfill({
    status: 200,
    json: [{ id: 1, name: 'Mock Product' }]
  })
);

// Simulate 500 error
await page.route('**/api/data', route =>
  route.fulfill({ status: 500, body: 'Internal Server Error' })
);
abort(errorCode?)Promise<void>
Aborts the request — simulates network failure or blocked resource.
typescript
// Block all image loading
await page.route('**/*.{png,jpg,gif}', route => route.abort());
continue(options?)Promise<void>
Lets the request proceed, optionally modifying url, headers, method, or postData before it goes to server.
typescript
// Add auth header to every request
await page.route('**', route =>
  route.continue({
    headers: { ...route.request().headers(), Authorization: 'Bearer token123' }
  })
);
network APIRequestContext HTTP client for API testing — accessed via request fixture or context.request
get / post / put / patch / delete(url, options?)Promise<APIResponse>
HTTP verbs for API calls. Each returns an APIResponse with status, body, headers. Options include headers, data (body), params (query string), timeout.
typescript
test('CRUD API test', async ({ request }) => {
  // GET
  const getRes = await request.get('https://api.example.com/users');
  expect(getRes.ok()).toBeTruthy();

  // POST with JSON body
  const postRes = await request.post('https://api.example.com/users', {
    data: { name: 'Ravi', email: 'ravi@test.com' }
  });
  expect(postRes.status()).toBe(201);

  // PUT — update
  await request.put('https://api.example.com/users/1', {
    data: { name: 'Ravi Kumar' }
  });

  // DELETE
  await request.delete('https://api.example.com/users/1');
});
fetch(url | Request, options?)Promise<APIResponse>
Generic fetch — use when you need full control over method, headers, body. Also accepts a Request object from a page.on('request') event.
typescript
const res = await request.fetch('https://api.example.com/data', {
  method: 'POST',
  headers: { Authorization: 'Bearer token' },
  data: { query: 'test' }
});
const body = await res.json();
dispose()Promise<void>
Disposes the context and all its responses. Only needed when you manually create a context via playwright.request.newContext(). The test-scoped request fixture auto-disposes.
typescript
const apiCtx = await playwright.request.newContext({
  baseURL: 'https://api.example.com'
});
// ... use apiCtx ...
await apiCtx.dispose(); // manual cleanup
test runner expect Assertion library — auto-retries until assertion passes or timeout

Assert element is visible (rendered and not display:none/visibility:hidden) or hidden. Auto-retries for up to 5s by default.
typescript
await expect(page.locator('.success-msg')).toBeVisible();
await expect(page.locator('.spinner')).toBeHidden();
toBeEnabled() / toBeDisabled()Promise<void>
Assert a button/input is enabled or disabled (has/lacks the disabled attribute).
typescript
await expect(page.locator('button#submit')).toBeEnabled();
await expect(page.locator('input#locked')).toBeDisabled();
toBeEditable()Promise<void>
Assert element is not readonly and not disabled — user can type into it.
typescript
await expect(page.locator('#name-field')).toBeEditable();
toBeEmpty()Promise<void>
For inputs: value is ''. For other elements: no child nodes/text.
typescript
await expect(page.locator('#search')).toBeEmpty();
await expect(page.locator('.cart-items')).toBeEmpty();
toBeFocused()Promise<void>
Assert the element has browser focus (is the active element).
typescript
await page.locator('#email').focus();
await expect(page.locator('#email')).toBeFocused();
toBeAttached()Promise<void>
Assert element is in the DOM (attached), even if not visible. Opposite of detached.
typescript
await expect(page.locator('#hidden-panel')).toBeAttached();
toBeChecked(options?)Promise<void>
Assert checkbox or radio is checked. Pass {checked: false} to assert unchecked.
typescript
await expect(page.locator('#terms')).toBeChecked();

toHaveText(text) / toContainText(text)Promise<void>
toHaveText = full text must match. toContainText = text contains the substring.
typescript
await expect(page.locator('h1')).toHaveText('Welcome Back');

toHaveURL(url)Promise<void>
Assert the page URL matches a string or regex.
typescript
await expect(page).toHaveURL(/dashboard/);
await expect(page).toHaveURL('https://app.com/home');

class MyReporter {
